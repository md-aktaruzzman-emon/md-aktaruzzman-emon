## Systems Programming

I have a strong interest in how software interacts with memory, operating systems, compilers, and hardware-level constraints.

**Key areas:**

- Manual memory management in C and C++
- Pointer arithmetic and memory layout understanding
- Stack, heap, and resource lifetime analysis
- RAII-based resource management in modern C++
- Debugging runtime issues with GDB
- Detecting memory leaks and invalid access with Valgrind
- Writing efficient algorithms with attention to CPU and memory usage
- Building clean command-line tools and low-level utilities
